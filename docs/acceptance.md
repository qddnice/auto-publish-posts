# auto-publish-posts 验收用例

## 后端

1. 执行 `mvn clean test`，确认素材选择、发帖状态机、违禁词、置信度规则通过。
2. 启动 PostgreSQL 和 Redis 后运行后端，确认 Flyway 在 `dev` schema 创建 `ap_*` 表。
3. 调用 `/api/auth/login` 获取 token，再访问配置、素材、发帖、回复和统计接口。

## 前端

1. 执行 `pnpm test`，确认状态映射测试通过。
2. 执行 `pnpm run type-check` 和 `pnpm run build`。
3. 打开前端页面，验证配置、运营工作台、素材与发帖、违禁词、统计页面可切换。

## 业务流程

1. 新增 FB 主页配置，启用自动回复和自动发帖。
2. 创建素材库和素材，生成 FB 发帖计划，审核通过。
3. 在运营工作台查看回复记录，确认 `MEDIUM` 置信度可标记待复查。
4. 人工回复前命中违禁词时阻止发送。
5. 执行紧急停止，确认返回影响配置数量和取消任务数量。
